How the weekly report works

  1. You upload two spreadsheets: the employee hours file (from payroll / time) and the contract hours file (from HR / Clockrite, or a simple table if you use that format).
  2. The app reads the employee file and builds one row per person, using Pay ID (same as Sage number / SageNo in the report).
  3. It reads the contract file and finds each person’s Payroll number and Contract hours (or columns in a single header row if you use a flat table export).
  4. It joins the two: Pay ID = Payroll number. If no payroll number is found, it may fall back to matching on name (check Contract match in the output).
  5. It calculates Overtime = total paid hours minus contracted hours, and splits employees into agency vs Gazebo categories for analysis.
  6. You can Download Excel for a full workbook (all data, not only the on-screen preview).

Employee hours file

What it is — your payroll or time system export of hours for the week (e.g. paysummary / gross / clock export).

Where to get it — export from the system you already use (e.g. payroll software or time & attendance) as .xls or .xlsx. The export must include a Pay ID column so each row can be matched; that ID is the same as the Payroll / Sage number in HR.

What we need in the file — at minimum: employee name, Pay ID, and the hour bands the report uses (basic, overtime, holiday, etc., depending on your file layout). If the layout changes, check that the Pay ID column is still present.

Contract hours file

What it is — the agreed contracted hours per person for comparison with paid hours.

Where to get it — from Clockrite (or the system that holds your contracts). Typical path: run an Employee details (or “Advanced”) type report and export to Excel, so each person shows a Payroll number and Contract hours (sometimes on separate label rows in the same block — the app supports that format).

What we need — each person’s Payroll number must match the Pay ID in the employee hours file. Contract hours of 0 is valid and still joins correctly for overtime maths.
